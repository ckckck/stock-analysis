# 全局规则

## 启动流程

- 每次对话开始时，必须首先使用 Skill 工具加载 `using-superpowers`，然后再回应用户。

## 语言与输出

- 回复、文档、工具参数、思考过程均使用中文；
- 代码、文件路径、技术术语可保留原文。
- 与版本协作相关的说明性文字使用中文：`git commit` 的标题与正文、推送或合并时附带的说明、合并/拉取请求（PR/MR）的标题与描述、Issue/讨论中的标题与正文（若使用）等；分支名、标签名、远程名及命令参数等可按项目或工具惯例保留英文。

## Amp 子代理选择

- 仅针对 Amp：当 Amp 进行代码审查时，不要使用 Review 子代理；应使用 Oracle 子代理。该规则只约束 Amp 的审查子代理选择。
- 仅针对 Amp：当 Amp 需要使用 Search、Librarian 或 View Media 时，应使用 Oracle 子代理替代。该规则只约束 Amp 的检索、代码库理解与媒体查看/分析工具选择。

## 危险操作（需用户确认）

执行以下操作前，必须主动说明风险并征得用户明确同意：

- **删除**：删除文件、目录、数据库记录
- **破坏性命令**：`rm -rf`、格式化、清空等
- **敏感配置**：修改 `.env`、密钥、凭据相关文件
- **系统级修改**：修改系统配置、PATH、hosts 等
- **版本控制**：`git push --force`、修改 `.git` 目录、重置历史
- **批量操作**：大规模重命名、批量替换、批量删除
- **不可逆操作**：数据库迁移、DROP TABLE、清空生产数据
